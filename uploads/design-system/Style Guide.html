Design System Style Guide

A live reference for every token, primitive, pattern and interaction state. Product-agnostic — applies to any operator-facing dashboard, admin panel or data-heavy app. Pair with tokens.css and DESIGN_SYSTEM.md.

Tooltips

Light, white-background tooltips — never dark. Always rendered to document.body via portal so they escape stacking contexts. Default width 280 px (clamps to viewport), 250 ms delay-in, no delay-out.

Popovers

Same surface treatment as tooltips, but clickable / hoverable with rich content (gauges, charts, breakdowns, action lists). Width 260–320 px, 16 px inner padding, 12 px radius, slightly stronger shadow than a tooltip.

Dropdowns

Portaled, position-fixed popups that flip upward when there isn't room below. Trigger has 3 states (rest / hover / open). Selected option uses slate-100 background. Grouped dropdowns add 10 px / 700 / slate-400 uppercase section headers.

Behaviors:
  • Trigger and popup are connected — popup matches trigger width by default.
  • Popup is portaled to document.body with position: fixed so it escapes overflow/stacking.
  • Auto-flips upward when there isn't ~290 px below.
  • Closes on outside click, scroll, or escape.
  • Section headers: 10 px / 700 / slate-400, uppercase, 0.6 tracking, top divider 1 px #F1F5F9.

Modals

Centered card with a slate-tinted overlay. Header has a 18 px / 700 title plus a 13 px muted subtitle. Footer is right-aligned with the destructive or cancel action on the left of the primary.
